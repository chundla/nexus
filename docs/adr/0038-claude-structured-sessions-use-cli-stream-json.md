# ADR 0038: Claude structured Sessions use the Claude Code CLI stream-json protocol

## Status
Accepted

Claude **Sessions** will use a structured primary **Session Surface** on local **Workspaces**, **Remote Workspaces**, and supported iPhone **Remote Client** paths. Nexus integrates only by spawning the `claude` executable with non-interactive print mode and bidirectional `stream-json` (`-p`, `--input-format stream-json`, `--output-format stream-json`), not the Claude Agent SDK and not a PTY-backed interactive terminal as the primary surface. Each **Session Record** maps 1:1 to one Claude session id stored as **Session Record** adapter metadata; relaunch resumes via `--resume` (or equivalent) on a long-lived child process. Permission prompts in the stream map to shared app-native **Approval Requests** with `--permission-mode default`; launch uses the **Workspace** target as cwd with workspace-scoped `--add-dir`, not `--bare` by default.

**Considered options:** (1) Keep Claude terminal-backed on all targets — rejected; blocks structured **Remote Client** operation, shared **Approval Requests**, and **Agent Turn** presentation parity. (2) One-shot `claude -p` per user prompt — rejected; breaks streaming, mid-turn approvals, and remote runner attach. (3) Agent SDK sidecar — rejected; duplicates the service’s CLI-shaped provider adapter model and complicates remote **Host** deployment. (4) Dual terminal + structured surfaces — rejected; two runtimes, IPC routes, and test matrices indefinitely.

**Consequences:** `ClaudeProviderModule` constructs a protocol runtime (local and remote) instead of `makeLocalTerminalRuntime` / `makeRemoteTerminalRuntime`. Remote Claude follows ADR 0031: tmux-backed detached runner, SSH stdio bridge, no terminal **Session Surface**. **Provider Health** adds a `stream-json` capability probe (ADR 0015 spirit). **Session Presentation** adds Claude **Agent Turn** feed segments (ADR 0037). Hard cutover removes Claude PTY/tmux UI and terminal-only remote routes; legacy terminal **Session Records** do not migrate tmux transcript. README, `ARCHITECTURE.md`, and milestone “Claude terminal-backed” text must be updated when implementation lands. Orphan tmux sessions on **Hosts** from the prior era are not auto-cleaned.
